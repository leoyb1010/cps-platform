## 第 0 章 · 一页看懂

**这是什么:** 一个以「订阅会员分销」为根,生长出来的**多品牌交易与资金清结算平台**。它把「卖谁的会员、谁来卖、卖了怎么分钱、出了投诉谁兜底」这套链路完整产品化了。

**现在能干什么(已落地):** 一个跑通了一个多亿流水的 CPS 投流业务,已经沉淀成一套软件系统——品牌接入、选品发单、代理商投流、订单与连续包月、清结算分账、商户号/号池风控、投诉工单售后、资金合规、数据归因,共 16 个功能模块、13 张数据表,带企业级 RBAC 与全链路审计。

**要往哪走(新方向,三条并行):**
1. **会员对赌互销网络** —— 品牌之间互相承诺销量、互相发单代卖,品牌足够多时演化成「自由组合、自由发单/接单、接单方自定对价」的交易市场。
2. **资源置换市场** —— 品牌之间拿广告资源、会员资源做置换,**互相开票、各自确认收入**,把「以物易物」做成可入账、可规模化的等值交换生意。
3. **AIGC 投放素材工厂** —— 挂在已有投放链路上,为投放自助生成图/视频素材(积分制),并回收真实转化数据反哺「什么素材更能带货」。

**统一底座:** 上面三条新业务 + 原有 CPS 自营,**全部复用同一套「清结算 + 风控 + 对账 + 履约」中台**。这套中台是把多方资金算到分不差、把承诺额度/对价/置换开票记成一本清账的能力,是整个平台真正的地基。

## 第 1 章 · 业务背景与原点

### 1.1 起点:CPS 投流业务

CPS(Cost Per Sale)投流,本质是:**品牌方(如有道)把会员订阅产品交给代理商去投广告卖,代理商自费买流量、做素材、跑广告,只有用户真的付费订阅了连续包月,品牌才按约定比例给代理商分钱。**

- **卖的是连续包月会员**,不是一次性会员。首月低价(如 29.9 元)拉新,用户不主动取消则按月续费。
- **真正看重的是 LTV(用户生命周期价值)**,不是首单赚多少 —— 续费率决定生死。
- **参与四方:** 品牌方(广告主+产品方)、代理商(投流方)、用户(付费会员)、投放平台(巨量/磁力/广点通/百度/支付宝等流量来源)。

### 1.2 现状与痛点

当前业务已经做到一个多亿流水(卖有道 30–50 元会员),但业务**结构单薄**:只有「自营投流」这一条腿。已经验证可行、且已建成系统能力的环节包括:
- 清结算分账已跑通(算到分不差);
- 投诉与风控必须由人工介入做服务和售后,这块也已跑通;
- 商户号有硬性约束:**投诉率、升级投诉率必须低于一定阈值**,否则商户号会被风控降权/熔断 —— 这是整个生意的合规生命线。

> 平台化的动机:把这套已经跑通的能力,从「只服务自己一条业务」扩展为「能承载多品牌、多种交易形态」的平台。

## 第 3 章 · 新方向(本版核心)

下面三条是要拓展的新业务。每一条都先讲**正确理解的业务定义**,再讲**它怎么复用现有能力**,再讲**需要新建什么**。

### 3.1 会员对赌互销网络(Co-Membership Cross-Sell Network)

**正确的业务定义(请其他 agent 以此为准,不要理解成 To C 打包卖会员):**

这不是「把多个品牌权益打包成一张卡卖给用户」。它是**品牌与品牌之间的互销对赌**:

- 最小单元是**双边对赌**:我向品牌 A 承诺「帮你卖 100 万的(你的)会员」,A 也向我承诺「帮我卖 100 万的(我的)会员」。双方各自利用自己的渠道/流量帮对方代卖,用承诺额度互相对赌。
- 当**接入的品牌足够多**时,双边对赌升级为**多边交易网络**:任何品牌都可以在网络里**自由组合、自由发单**(我要找人帮我卖 X)、**自由接单**(我愿意帮你卖 Y);**接单的品牌自己决定接受什么对价**(按 CPS 分成?按承诺量互换?按固定采买?由接单方定)。
- 平台角色:**撮合 + 记账 + 履约兑付**。平台不替品牌做履约承诺,而是把「谁承诺帮谁卖多少、卖到没卖到、该兑付多少对价」记成一本可对账的清账。

**怎么复用现有能力:**
- 「品牌管理 + 选品市场 + 订单 + 清结算 + 数据归因」几乎直接可用:发单/接单走选品市场,代卖产生的订单走订单流,分成/对价兑付走清结算,完成度看归因。
- 互销带来的跨品牌订单,正好用现有「多方对账 + 逆向冲账」来保证不串账、不重复结算。

**需要新建的能力:**
- **承诺额度台账**:记录每对品牌互相承诺的销售额度、已完成额度、缺口与到期处理。
- **对赌结算规则引擎**:支持多种对价形态(分成 / 承诺量互换 / 固定价),接单方自定义对价并锚定到结算。
- **自由发单/接单市场**:从「品牌↔代理」的撮合,扩展为「品牌↔品牌」的多边撮合与匹配。
- **完成度与违约处理**:承诺没完成时的差额补偿、信用分扣减、再撮合机制。

### 3.2 资源置换市场(Resource Barter & Cross-Invoicing Market)

**正确的业务定义(请其他 agent 以此为准,不要理解成单纯以物易物):**

资源置换的本质是**「互相开票、各自确认收入」的等值交换生意**:

- 双方各自拿出资源做置换 —— 典型是**广告资源**(开屏/信息流/曝光位)与**会员资源**(会员权益/账号/激活码),或广告换广告、会员换会员。
- 关键不在「换东西」,而在**财务处理**:双方**互相开具发票、各自把这笔置换确认为收入**。也就是把一次资源对换,做成两笔真实、合规、可入账的收入交易 —— 这让置换从「人情往来」变成「可规模化、可计量、能进财报」的业务。
- 平台角色:**资源估值 + 等值撮合 + 双向开票与收入确认台账**。

**怎么复用现有能力:**
- 「品牌管理 + 清结算 + 资金合规 + 审计」是天然底座:资源挂牌走品牌侧,等值核算/开票确认走清结算与合规,全程留痕走审计。
- 现有「对账恒等式 + 资金流向校验」可以直接用来保证「换出价值 = 换入价值 = 双向开票金额」三者自洽。

**需要新建的能力:**
- **资源目录与估值**:把广告位、会员权益等资源标准化挂牌,给出可比的估值口径(CPM/权益单价等)。
- **等值置换撮合**:撮合双方达成等值(或带差额找补)的置换组合。
- **双向开票与收入确认台账**:记录每笔置换对应的双方发票、确认收入金额、税务口径、对账状态。
- **差额结算**:置换不等值时,用现金或额度补差,接回清结算。

### 3.3 AIGC 投放素材工厂(Creative Factory,第二曲线)

**业务定义:** 在已有投放链路上,挂一个**按量计费(积分制)的 AIGC 素材生成**模块,让品牌/代理**自助生成**投放用的图/视频素材,平台只提供工具与算力,不提供人力外包。

- 合规外壳:在中国不直接转售算力 token,而以「软件服务 + 积分消耗」的形态计费(行业标准做法)。
- **差异化闭环:** 平台手里有真实投放渠道与转化数据 —— 生成的每条素材都能接入 CPS 投放链路,回收真实点击率/转化率,反过来沉淀「什么素材更能带货」。这是纯 AIGC 工具平台拿不到的数据闭环。

**怎么复用现有能力:** 直接挂在「投放 → 订单 → 归因」链路上;积分计费复用清结算的钱包/账务能力。

**需要新建的能力:** 积分钱包、素材生成工作流与模板、素材↔转化数据的归因回流。

## 第 4 章 · 整体业务结构

```
                  ┌───────────────────────────────────────────────┐
   业务层(对外)   │  ① CPS 自营投流(现金牛,已跑通一个多亿流水)        │
   多种交易形态    │  ② 会员对赌互销网络(品牌↔品牌 互销/对赌/自由发接单) │
   并行共存        │  ③ 资源置换市场(广告/会员资源置换 + 双向开票确认收入)│
                  │  ④ AIGC 投放素材工厂(积分制 + 转化数据回流)         │
                  └───────────────────────────────────────────────┘
                              ↓  全部复用同一套底座  ↓
                  ┌───────────────────────────────────────────────┐
   中台(地基)     │  清结算 + 风控 + 对账 + 履约 中台                  │
   真正的壁垒      │  分账引擎 · 多方对账 · 准备金 · 逆向冲账            │
                  │  承诺额度台账 · 对价/置换记账 · 双向开票确认         │
                  │  号池风控(投诉率/升级投诉率阈值) · 投诉工单售后     │
                  │  幂等/并发/审计闭环 · RBAC · 多租户隔离             │
                  └───────────────────────────────────────────────┘
                              ↓  支撑  ↓
                  ┌───────────────────────────────────────────────┐
   底层资源       │  账户体系 · 积分钱包 · 商户号/支付通道 · 合规资质     │
                  └───────────────────────────────────────────────┘
```

**一句话:** 上层是「多种把会员/资源变现的交易形态」,中层是「把这些交易算清楚、兜住风险的中台」,底层是「账户、钱包、商户号、合规」。新业务全部是中台的「应用」。

## 第 6 章 · 合规与风控红线(贯穿所有业务,不可逾越)

无论做哪条业务,这几条是硬约束,任何创意/优化方案都必须遵守:

1. **商户号阈值红线:** 投诉率、升级投诉率必须低于约定阈值,否则商户号被降权/熔断 —— 直接掐断收款能力。所有新业务都不能把投诉率推高。
2. **投诉/售后必须有人工介入:** 这是已跑通的服务,不能为了规模化而砍掉人工售后。
3. **资金合规:** 置换的双向开票、对赌的对价兑付、积分计费,都必须可入账、可审计、口径自洽(对账恒等式)。
4. **连续包月合规:** 自动续费的告知、取消、退款必须合规,否则放大投诉与舆情。
