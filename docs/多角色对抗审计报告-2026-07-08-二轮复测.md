## 试用角色

覆盖 8 类角色：平台超管、财务、风控/客服、运营、只读审计、品牌方、代理商、品牌审计。

每轮执行 33 个 API 对抗用例和 3 组浏览器截图，重点模拟：

1. 品牌/代理在客服门户中抢先关闭工单、冻结 SLA。
2. 有道合作方使用错误 `custId` 但正确 `merchantId` 和 RSA 私钥下单/查单。
3. callback URL 用 `nip.io/sslip.io`、保留网段、localhost、内网 IP 绕过 SSRF 防护。
4. 号池新增孤儿商户号、非平台角色新增商户号、权重越界。
5. 公开模拟支付端点在默认环境和生产误配环境下是否可置 `paid`。
6. 品牌门户登录、开发者中心、C 端市场移动端是否产生控制台错误或错误边界。

## Round 1 发现

| 严重度 | 域 | 问题 | 影响 |
|---|---|---|---|
| P1 | 客服 | 品牌门户可直接把工单置为 `resolved` 并将 `slaLeftMin` 置 0 | 外部客户可绕过平台风控关闭投诉，SLA 和客服队列失真 |
| P1 | 客服 | 代理门户同样可直接关闭工单并冻结 SLA | 代理可自证处理完成，影响退款/追责流程 |
| P1 | 支付/外部接口 | 有道接口只按 `merchantId` 找凭证，未绑定 `custId` | 同一 `merchantId` 私钥持有者可携带错误 `custId` 下单 |
| P1 | 支付/外部接口 | 有道订单查询可用错误 `custId` 查询同 `merchantId` 订单 | 合作方身份边界不完整，接口契约与文档不一致 |
| P1 | callback | callback URL 只做字面 host/IP 检查，未做 DNS 解析 | `127.0.0.1.nip.io` 一类域名可绕过内网回调限制 |
| P2 | 门户前端 | 品牌门户登录后触发内部后台数据水合，浏览器出现 `/brands /agents /orders /settlements /config /tickets` 403 | 门户账号体验噪声，且暴露内部接口路径 |

Round 1 结果：`findings=6`，API 用例 33，浏览器截图 3。

## 修复执行

1. 客服闭环：品牌/代理回复接口拒绝 `status=resolved`，只允许提交协作说明或流转到 `processing`；前端按钮从“标记已解决”改为“提交协作处理”。
2. 有道鉴权：`authn` 改为按 `merchantId + custId + active` 查找凭证；下单、退款、解约、查单统一要求 `custId` 与凭证绑定。
3. callback 防护：`validatePublicCallbackUrl` 改为异步 DNS 校验，阻断 localhost、内网 IP、保留/文档/benchmark 网段、域名内嵌内网 IP；保存、健康检查、webhook 投递和重投共用同一校验。
4. 模拟支付：`ENABLE_MARKET_SIM_PAY=true` 只在非生产环境生效，生产环境误配也不能公开置 `paid`。
5. 门户体验：真实模式登录后仅平台账号水合内部控制台 store；品牌/代理账号清理旧后台缓存但不拉内部后台接口。
6. 本地联调：CORS 同时兼容 `CORS_ORIGIN` 与 `CORS_ORIGINS`；fake-IP DNS 兼容只通过 `ALLOW_FAKE_DNS_CALLBACK_RESOLUTION=true` 在非生产联调中显式使用。
7. 测试补强：新增客服关闭拒绝、协作处理成功、`custId` 错配下单/查单拒绝、callback DNS 绕过拒绝、生产模拟支付误配拒绝、门户账号不水合内部 store。

## Round 2 复测

修复后清空审计 SQLite 库、重新 seed、重启后端和前端，再用同一脚本复测。

Round 2 结果：`findings=0`，API 用例 33，浏览器截图 3。

关键复测结果：

- 品牌/代理发送 `status=resolved` 返回 `ok:false`，工单未变为 `resolved`。
- `cust_attacker + mch_youdao` 的有道下单和查单均返回非 0 错误码。
- `http://127.0.0.1`、`https://192.168.1.10`、`https://127.0.0.1.nip.io` 均被拒绝。
- `https://example.com/youdao/callback` 在本地 fake-DNS 显式兼容开关下可保存；生产默认不放行保留网段解析。
- 模拟支付默认关闭；生产环境即便设置 `ENABLE_MARKET_SIM_PAY=true` 也拒绝置 `paid`。
- 品牌门户登录和开发者中心浏览器检查无 401/403 控制台错误，无错误边界。

## 余留风险

callback 当前是在保存/投递前做 DNS 解析校验，已覆盖本次 SSRF 绕过路径。若进入更高安全等级的生产环境，建议进一步做“解析结果与实际连接 IP 绑定”的出站 HTTP 客户端，降低 DNS rebinding 的时间窗。
