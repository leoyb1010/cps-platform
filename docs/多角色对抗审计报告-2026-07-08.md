## 修复前发现

1. **P0：公开模拟支付端点可无鉴权置 paid**
   - `POST /market/bundle/:id/pay` 无需登录、无需支付平台签名或回调验真即可把套餐置为 paid。
   - 风险：生产暴露时可绕过真实支付确认。

2. **P0：外部投诉脏数据可打崩客服台**
   - `POST /complaints/ingest` 允许写入不存在品牌，且新工单状态为前端不识别的 `open`。
   - 风险：客服 SLA 队列、退款、转派、关闭动作不可用。

3. **P1：品牌回调 URL 存在 SSRF/内网探测风险**
   - 品牌可配置 `http://127.0.0.1/...`，健康检查由服务端发起 POST。
   - 风险：可探测 localhost、内网或云元数据地址。

4. **P1：号池允许孤儿商户号**
   - `POST /merchants` 可录入不存在品牌的商户号。
   - 风险：号池路由、品牌隔离、报表归属被污染。

## 已完成修复

1. 模拟支付默认关闭，仅显式设置 `ENABLE_MARKET_SIM_PAY=true` 的测试/沙箱环境可使用。
2. 外部投诉接入校验品牌和代理存在，新工单状态统一为 `pending`。
3. 客服台对未知 `source/status/level` 做 UI 兜底。
4. 回调 URL 保存、健康检查、webhook 投递三处增加公网 HTTPS 校验，阻断 localhost/私网/metadata 地址。
5. 号池新增商户号校验品牌存在，拒绝孤儿商户号。
