## 修复清单

### 1. 资金与状态机

- 退款冲账与代理回收的余额不足回退改为 CAS 循环，只有真正写入成功的金额才能作为 `share`/`deducted` 继续记账。
- 准备金 clawback 使用 `status + amount` 条件认领；竞争输家重新读取，`reserveClawedBack` 与 `frozen` 只记录真实追回额。
- 补扣先在事务内把 `pending` 认领为 `processing`，再用 `active + currentPeriod` 条件推进；解约或正常续扣先提交时不再创建订单。
- 解约状态、补扣取消、Subscription churn、审计与 webhook outbox 同事务提交；提交后立即投递，崩溃时由 pending sweep 接管。
- `Order.settlementId` 升级为带 FK 的原账期关系；退款只冲原结算单，不再取品牌最新一期。无法唯一识别的旧数据拒绝猜测，避免错账。
- 存量 `agentShareSnapshot=0` 在迁移中按 `agentPayout + reversal` 回填，后续退款比例不再递减漂移。

### 2. 认证、签名与 SSRF

- Refresh token 在事务内以 `revoked=false` 条件原子认领；并发或重复使用触发整族吊销和 tokenVersion 提升。
- 有道三个写接口强制 `nonce`；RSA 验签通过后，以 `merchantId + custId + nonce` 哈希写入 DB 唯一键，跨进程/跨实例拒绝重放。
- 首登强制改密改为后端门禁；改密前只允许 `/auth/me`、`/auth/change-password` 与退出流程。
- `ALLOW_WEAK_SECRETS` 在生产环境彻底失效；开发环境也只允许弱值，不允许空值，避免启动成功后登录 500。
- Callback guard 完整拒绝 IPv4-mapped IPv6、非 global-unicast、链路本地、组播与文档网段。
- 实际 HTTPS 请求固定到校验过的 IP，保留原域名 SNI/证书校验且不跟随重定向，关闭 DNS rebinding TOCTOU。

### 3. Real 模式与权限

- 新增品牌专用 `/portal/brand/candidates`，资源置换不再依赖代理商权限端点。
- 品牌订单接口改为游标分页；前端聚合到安全上限后明确告警并禁用不完整 CSV。
- 平台订单截断时停止计算误导性比率并禁用对账导出。
- 服务端明确返回 `permissions: []` 时不再回退本地角色默认权限；品牌/代理每条路由增加权限守卫。

### 4. 工程与部署

- Compose server healthcheck 和 Docker CI 从 `/health` 改为检查数据库的 `/ready`。
- 清零 24 个 ESLint error 与 3 个 warning，移除 `continue-on-error`，lint 现在阻断 CI。
- Settlement 测试统一使用 `/tmp` 隔离 SQLite helper，带 PID 命名与安全 cleanup；初始化失败不再触发二次清理异常。
- SQLite/PostgreSQL 两份 schema、迁移和 OpenAPI 同步更新。
