## 0. 现状体检结论（grounded，非泛泛）

逐项扫描后，当前还存在的可打磨点（均非新功能）：

| # | 现状 | 影响 |
|---|---|---|
| a | **表格无粘性表头、无斑马纹** | 长表格滚动时列错位、行难追踪 |
| b | **无加载/骨架态** | 现在数据是即时 mock，体感没问题；但任何异步动作（结算、退款、保存）点下去到 toast 之间**无过渡反馈**，显得"卡一下" |
| c | **可点卡片/行只有变色，无"浮起"层次** | 缺少高级感的深度反馈 |
| d | **KPI 数字只在挂载时滚动**，store 变化时数值**瞬间跳变** | 退款后待结算等数字硬跳，不连贯 |
| e | **0 处 aria-label、Drawer/Modal 无 role=dialog** | 纯图标按钮（铃铛/关闭/重播）对读屏不可用；无障碍不达标 |
| f | **空筛选结果**：部分表格筛到 0 行时直接空白（EmptyState 仅 4 处） | 看着像"坏了" |
| g | **滚动条样式仅 webkit**，内层滚动区（抽屉/下拉/订单流）未统一 | 跨区域观感不一致 |
| h | **useStore 返回整树**，写入时所有消费组件重渲染；几乎无 memo | 当前规模无感，但表格变长 / 实时流加密后会掉帧 |
| i | **品牌深色小方块（有/喜/芒）** 在浅色行里偏重 | 视觉密度偏高（你之前提过黑底白字） |
| j | **字号/间距/圆角**多处硬编码像素（text-[12.5px] 等），缺统一比例尺 | 跨页面细微不一致 |

## 1. 本轮五条线 + 具体任务

### 线 A · 质感细节（最直接的"高级感"）
- ★ **表格粘性表头**：`TableShell` 的 `<thead>` 加 `sticky top-0` + 毛玻璃底，长表滚动时表头常驻。
- ★ **斑马纹 + 行悬停浮起**：偶数行极浅底；可点行 hover 加 `hover:shadow` 微阴影 + 1px 描边加深，制造"浮起"。
- ★ **可点卡片浮起**：Dashboard 风险条 tile、品牌/代理行、选品卡片 hover 时轻微 `shadow` + `-translate-y-[1px]`（GPU 友好）。
- ☆ **品牌 mark 减重**：深色方块改为"浅底 + 品牌色字"或描边款，降低视觉重量，与整体浅色调统一。
- ☆ **分隔与留白节奏**：统一卡片内边距、区块间距到一套尺度（见线 E）。

### 线 B · 体感性能（让"等待"不突兀）
- ★ **异步动作的瞬时反馈**：结算/退款/保存等按钮点击后，按钮进入 `loading`（禁用 + 转圈 120–400ms 模拟），再出 toast。消除"点了没动静→突然弹"的断层。
- ★ **数字平滑过渡**：`CountUp` 支持"值变化时从旧值滚到新值"（不只挂载时）。退款后待结算、信用分等**滚动变化**而非硬跳。
- ☆ **骨架屏组件**（`Skeleton`）：为表格/卡片做骨架态，接真实后端时即可用；现在可用于"刷新/重置数据"的 200ms 过渡。
- ☆ **列表进入动画收敛**：实时流/活动流新条目用一致的轻微滑入，避免突现。

### 线 C · 健壮性 / 边界态
- ★ **全表格空态兜底**：任何筛选 / 搜索结果为 0 时统一渲染 `EmptyState`（带"清除筛选"动作）。
- ★ **数值/除零防御**：`Math.max(...[])`、`/ total` 等在空数组时的兜底（号池/品牌全空时不报 NaN/Infinity）。
- ☆ **localStorage 容错**：写入失败（隐私模式/超额）时降级为内存态并提示，不白屏。
- ☆ **路由兜底**：未知 hash 路由 → 重定向到经营总览（现在可能空白）。

### 线 D · 可访问性（A11y，达标 + 顺带提质感）
- ★ **图标按钮 aria-label**：铃铛、关闭、重播、菜单、搜索等纯图标按钮补 `aria-label`。
- ★ **Drawer/Modal 语义**：加 `role="dialog"` + `aria-modal` + `aria-labelledby`，并在打开时**初始聚焦**、关闭时**焦点归还**触发元素。
- ☆ **键盘可达**：可点 `<div>` 行补 `tabIndex`/`role` 或改 `<button>`；Segmented/Tabs 支持方向键。
- ☆ **对比度核查**：ink-4/ink-5 在浅底上的小字对比度过一遍（WCAG AA）。

### 线 E · 设计系统一致性（地基收口）
- ★ **排版比例尺收口**：把散落的 `text-[12.5px]/[11px]/...` 收敛为 4–5 档语义级别（如 `text-meta/label/body/title`），统一行高与字重。
- ☆ **间距/圆角令牌化**：卡片内边距、区块 gap、圆角统一引用 token，消除逐页微差。
- ☆ **颜色一致性复核**：UI 强调色 `#f5333b` 与有道官方红 `#FC011A` 的取舍统一（要么全站对齐 logo 红，要么明确分工：logo 用官方红、UI 用柔和红——当前是后者，确认即可）。
- ☆ **图表视觉统一**：各页自制 SVG 图（折线/柱/环/迷你）的描边宽度、网格色、字号统一为一套。

## 2. 优先级与分批（建议）

**P0（这批做，体感提升最明显，约 0.5–1 天）**
1. 表格粘性表头 + 斑马纹 + 行/卡片悬停浮起（线 A 前 3）
2. 异步动作 loading 反馈 + 数字平滑过渡（线 B 前 2）
3. 全表格空态兜底 + 数值除零防御（线 C 前 2）
4. 图标按钮 aria-label + Drawer/Modal role=dialog/聚焦归还（线 D 前 2）

**P1（紧接其后）**
5. 品牌 mark 减重、骨架屏组件、localStorage 容错、路由兜底
6. 排版比例尺收口（线 E ★）

**P2（锦上添花）**
7. 键盘方向键、对比度核查、间距/圆角令牌化、图表视觉统一、颜色一致性复核

## 3. 验收口径（每条都要可验证）
- 构建：`tsc -b` + 严格 `--noUnusedLocals` + 生产构建 **全过**，控制台 **0 报错**。
- 体感：表头滚动常驻；行/卡片 hover 有缓动浮起；结算/退款点击有即时 loading；退款后数字**滚动**变化。
- 健壮：任意筛选到 0 行有空态；号池/品牌清空不出 NaN；未知路由不空白。
- A11y：读屏可读出每个图标按钮；Tab 焦点环可见、弹层聚焦归还；关键文本对比度 ≥ AA。
- 一致性：随机抽 3 个页面，卡片内边距/字号档位一致。

## 4. 明确不做（守住"不加功能"）
- 不加新页面、新业务动作、新数据维度。
- 不接真实后端 / 支付 / 分账（那是 L3，另立项）。
- 不做暗色模式（属新主题，非本轮"打磨"范畴；如要可单列）。
- 不改业务口径与指标定义（已锁定在字典）。
