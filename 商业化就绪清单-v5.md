## 二、经营 / 资质 / 资金合规就绪度（代码替代不了 → 必须线下办）

> 这部分是 CPS 会员清结算平台「真正能对外收钱、合法经营」的硬门槛。**没有任何一项能靠前后端代码绕过。**

| 必办项 | 为什么代码替代不了 | 谁来办 | 阻塞级别 |
|---|---|---|---|
| **持牌分账 / 支付通道签约** | 平台不得碰资金本体（二清红线）。必须接**持牌机构**（连连/汇付/微信官方分账/银行存管）做分账，需企业资质签约 + 风控审核 | 你方公司 + 持牌机构 | 🔴 收款必备 |
| **ICP 经营许可证 / EDI** | 经营性互联网平台的法定准入 | 你方 + 通信管理局 | 🔴 上线必备 |
| **网络安全等级保护（等保 2.0）** | 处理大量用户/交易数据的合规要求（通常二级起） | 你方 + 测评机构 | 🔴 合规必备 |
| **资金合规法律意见书** | 分账模式、代理佣金、灵活用工开票是否触红线，需律师出具 | 你方 + 律所 | 🔴 风险必办 |
| **个人信息保护合规（《个保法》）** | 用户数据收集/存储/出境的合法性基础，隐私政策、最小必要、加密脱敏的法律落地 | 你方 + 法务 | 🔴 合规必办 |
| **真实风控数据源** | 投诉率/黑产/设备指纹等需接真实数据（黑猫、12315、风控厂商、支付风控回传），demo 的阈值逻辑要喂真数据才有意义 | 你方 + 数据/风控厂商 | 🟠 风控生效必备 |
| **品牌方正式合作协议 + 真实费率/结算条款** | 联运分发的法律与商业基础，决定真实分润规则 | 你方 BD | 🟠 业务必备 |
| **灵活用工 / 众包开票通道** | 个人代理佣金合规发放（规避无票打款税务风险），需签约平台 | 你方 + 用工平台 | 🟠 代理结算必备 |
| **生产部署与安全审计** | 真实域名/HTTPS 证书/WAF/密钥管理/渗透测试，secret 不能是 demo 值 | 你方运维 + 安全 | 🟠 上线必备 |

> 我（AI）在本工程中**已主动拒绝**实现任何「刷单 / 拓新订单生成、低价转正价规避支付风控」之类功能 —— 这类属虚假交易（违反《电子商务法》第十七条、《反不正当竞争法》第八条）并规避支付平台风控，不能写进系统。合规页 `/compliance` 已将「虚假交易 / 刷单冲量」列入"绝不能做"。这条边界不因任何要求而改变。
